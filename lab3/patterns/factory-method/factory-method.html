Factory Method Pattern Class Diagram

The Factory Method pattern is a creational design pattern that provides an interface for creating objects in a superclass, but allows subclasses to alter the type of objects that will be created.

In our implementation, we have an abstract UserFactory class that defines the factory method createUser(), and a ConcreteUserFactory that implements this method to create different types of users based on their type.

Implementation Notes

Our implementation includes:

  • An abstract UserFactory class with the abstract factory method createUser()
  • A concrete factory class ConcreteUserFactory that creates different user types
  • A User interface that defines common methods for all user types
  • Concrete user classes (AdminUser, CustomerUser, ManagerUser) that implement the User interface
